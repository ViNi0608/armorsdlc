2. Configuration Issues Detected
Issue: Public S3 Bucket
Resource: s3://user-backups
Severity: Critical
Fix: Set bucket access to private using AWS IAM policies or S3 block public access.
References:
  • AWS: Block Public Access to S3
  • OWASP Cloud-Native Security Top 10
Issue: Overly Permissive IAM Policy
Resource: AdminPolicy
Severity: High
Fix: Follow the principle of least privilege — remove wildcard actions and unused services.
References:
  • AWS IAM Best Practices
  • OWASP: IAM Risks in Cloud
Issue: SSH Port Open to Public
Resource: 0.0.0.0/0 on Port 22
Severity: High
Fix: Limit SSH access to trusted IPs, use VPN or jump hosts, disable root login.
References:
  • AWS EC2: Key Pairs & SSH
  • SSH Security Best Practices
